A. Data Preparation¶

  • Dataset : https://github.com/MWiechmann/enron_spam_data Enron Spam Dataset has a total of 33,716 e-mails total with 17,171 spam and 16,545 non-spam ("ham") e-mail messages. It has 5 columns available, 'Message ID', 'Subject','Message','Spam/Ham' and 'Date'.

  • Few of the examples in the dataset have NaN values in the subject and message field. This is countered by dropping the examples where message and subject both are NaN.

  • Subject and Message are combined to create the email text which helps to add more features for the model.

  • This dataset has equal representation across both the classes, which makes it a suitable option for classification task with the objective to classify an new email as spam or ham based on the message.

from sklearn.feature_extraction.text import CountVectorizer

emails['text'] = emails['Subject'].fillna('') + ' ' + emails['Message'].fillna('')
vectorizer = CountVectorizer(stop_words='english')
X = vectorizer.fit_transform(emails['text'])

total_term_count = X.sum()  # total number of term occurrences across all documents
term_frequencies = np.asarray(X.sum(axis=0)).flatten()
terms = vectorizer.get_feature_names_out()

term_freq_df = pd.DataFrame({
    'Term': terms,
    'Rank': 0,  # filled below
    'Frequency': term_frequencies,
    'Probability': term_frequencies / total_term_count  # p(t) = f(t) / sum of all term counts
})

term_freq_df = term_freq_df.sort_values(by='Frequency', ascending=False).reset_index(drop=True)
term_freq_df['Rank'] = term_freq_df.index + 1

top5 = term_freq_df[['Term', 'Rank', 'Frequency', 'Probability']].head(5)

B2.2.3 Regression Line¶

The fitted line as seen below from the plot showing the observed data with predicted data.

  • The log-log plot follows roughly a linear pattern across most of the rank range. Deviations mostly occur at low ranks. The fitted regression coefficient -1.45 is off the ideal value -1.0. This indicates the distribution is more skewed than classic Zipf. The regression lins fits the middle rank range wail but under fits at extremes.

# B.2.3. Linear regression on log(p_t) ~ log(k_t) to estimate Zipf exponent
from sklearn.linear_model import LinearRegression

# x = log(rank), y = log(probability) — fit directly, no double-log
x = np.log(term_freq_df['Rank'].values).reshape(-1, 1)
y = np.log(term_freq_df['Probability'].values)

lm = LinearRegression()
lm.fit(x, y)
y_pred = lm.predict(x)

coef = lm.coef_[0]
print(f"Regression coefficient (Zipf exponent): {coef:.4f}")
print(f"Intercept: {lm.intercept_:.4f}")
print(f"Interpretation: p(t) ∝ 1 / k^{abs(coef):.2f}")

# Plot data and regression line on log-log scale
plt.figure(figsize=(8, 5))
plt.loglog(term_freq_df['Rank'], term_freq_df['Probability'], alpha=0.5, label='Observed')
plt.loglog(term_freq_df['Rank'], np.exp(y_pred), 'r--', linewidth=2, label=f'Regression (slope={coef:.2f})')
plt.xlabel('Rank $k_t$ (log scale)')
plt.ylabel('Probability $p_t$ (log scale)')
plt.title("Zipf's Law: Probability vs. Rank")

C3. Terms and Conditional Probabilities¶

  • Two words chosen are t1=prize; t2= meeting which are relevant to the class c1=spam; c2=ham respectively.

  • The probabilities p(t1/c1), p(t1/c2), p(t2/c1) and p(t2/c2) represent the number of message that were marked to the class (c1/c2) when the target word (t1/t2) was present in the message. They provide the information that how likely is a word to be present in the message when it belongs to a class.

  • The probability values are sensible based on the common email patterns. The word prize is more likely to appear in a email which is spam whereas you would expect the word meeting to appear more likely in a non-spam email.

  • These values are particularly useful in classification task as they difference between P(word/spam) and P(word/ham) for the terms show a clear indication as to which class email belongs to.

D. Classification¶

D.1 Probabilistic Naive Bayes Model (Bernoulli)¶

  • Depending on the task and objective of the model any of the 4 values from the evaluated confusion matrix can vary drastically,

    • Minimize False Negatives -> In critical scenarios where decisions may have life-threatening consequences, it may be acceptable to have false alarms if it ensures the model does not miss an actual positive case for eg. screening a medical device for equipment failure.

    • Minimize False Positives -> In some scenarios it can be more expensive to have false alarms. For eg in case of spam detection a false positive could mean the end user may miss an important email.

    • Minimize both FP and FN (Maximize F1 Score) -> Tasks with balanced datasets where both types of errors can have similar consequences. It might be preferable to maximize the F1 score of the model which is a measure of how good the models precision and recall is which directly affects to minimize False positives and False Negatives.

Hence to compare the performance of the model here with different values of alpha I have used F1 score in addition to accuracy and kappa.

In [37]:
# D.1. Bernoulli Naive Bayes classification
from sklearn.naive_bayes import BernoulliNB
from sklearn.metrics import accuracy_score, cohen_kappa_score, f1_score, confusion_matrix

results = []

for alpha in [0.001, 1.0]:
    bnb = BernoulliNB(alpha=alpha, binarize=0.0)
    bnb.fit(X_train_vec, y_train)
    y_pred = bnb.predict(X_test_vec)

    accuracy = accuracy_score(y_test, y_pred)
    kappa = cohen_kappa_score(y_test, y_pred)
    f1_spam = f1_score(y_test, y_pred, pos_label='spam')
    results.append({'Model': f'Naive Bayes alpha={alpha}', 'Accuracy': accuracy, "Cohen's Kappa": kappa, 'F1 Score (spam)': f1_spam})

    cm = confusion_matrix(y_test, y_pred, labels=bnb.classes_)
    fig, ax = plt.subplots(figsize=(5, 4))
    sns.heatmap(cm, annot=True, fmt='d', cmap='Blues',
                xticklabels=bnb.classes_, yticklabels=bnb.classes_, ax=ax)
    ax.set_xlabel('Predicted')
    ax.set_ylabel('Actual')
    ax.set_title(f'Confusion Matrix - BernoulliNB (alpha={alpha})')
    plt.tight_layout()
    plt.show()

    print(f"alpha={alpha}  Accuracy={accuracy:.4f}  Kappa={kappa:.4f}  F1(spam)={f1_spam:.4f}\n")

# Summary table
results_df = pd.DataFrame(results)

The above table shows the scores of the best model from each category. Non-linear Multi-layer Perceptron model achieves the best score overall in all the three metrics tracked. MLP model achieves an accuracy of 0.9898, which is slightly better than naive bayes model. The MLP model may benefit from additional context of subject + email in the training samples.

Naive Bayes model assumes each word is conditionally independent give the class. While this is not necessarily true, often the presence of some words (prize, free, won) mean the email is spam regardless of context making the assumption reasonable. Low alpha (0.001) achieves the better scores than higher alpha values as the data is not sparse.

The Perceptron model underperforms because it is sensitive to order of training samples and the non-convergence is caused by the overlap between two classes.

Future Work¶

  • Adding more and diverse training examples could help improve all models.
  • Applying TF-IDF weighting instead of raw counts could help reduce the dominance of specific terms towards a particular class.
  • Additional work on the data could include adding more classes instead of the current binary range of spam/ham. More labels like marketing/promotion, fishing, suspicious, valid etc. can help in distribution of classes and make the classification task easier.
